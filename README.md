# AmneziaWG Quickstart

Свой VPN на собственном VPS за 2 минуты. Без подписок, без логов, без посредников.

**AmneziaWG** = WireGuard + обфускация трафика. Под DPI выглядит как случайный шум по UDP-443 (тот же порт, что и HTTPS). Работает там, где обычный WireGuard и OpenVPN блокируются: РФ, Иран, Китай.

## Что получится

- Сервер VPN на твоём VPS, слушает `443/UDP`.
- Подключение с iPhone, Android, Windows, macOS, Linux через приложение **AmneziaVPN**.
- Команды управления одной строкой:
  ```
  sudo amneziawg-add-peer my-laptop      # добавить устройство
  sudo amneziawg-remove-peer my-laptop   # удалить
  sudo awg show                          # кто подключён, сколько трафика
  ```

## Что нужно

1. **VPS** на Ubuntu 22.04 или 24.04. Подойдёт самый дешёвый: 1 CPU / 1 GB RAM / 10 GB диска. Хостеры: Hetzner (~$4/мес), Aeza, FirstByte, любой другой.
2. **Локация VPS** там, где нужный тебе интернет: Германия, Нидерланды, США, Финляндия. Не в РФ.
3. **5 минут времени** и SSH-доступ к серверу под `root`.

## ⚠️ Перед началом — 3 правила

1. **Все команды ниже вводи на VPS-сервере** (после `ssh root@...`), а не на своём ноутбуке. Исключение явно отмечено: «на своём компе».
2. **На промптах скрипта вводи только то, что просят**: число для порта, короткое имя для устройства. Не путь к файлу.
3. **Файлы `.conf` и `.png` создаются скриптом сами.** Тебе ничего вводить вручную в `/root/vpn-clients/` не нужно.

## Установка — 3 команды

Зайди на свой VPS по SSH (`ssh root@<ip-сервера>`) и выполни **по очереди**:

```bash
apt update && apt install -y git
git clone https://github.com/serenkiy-commits/amneziawg-quickstart.git
sudo bash amneziawg-quickstart/install.sh
```

### Что ты увидишь после третьей команды

Скрипт распечатает заголовок и задаст 2-3 вопроса. Если кратко — на все вопросы можно **просто жать Enter** и получишь рабочую установку. Подробнее:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  AmneziaWG Quickstart
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Внешний интерфейс: eth0
  Публичный IP:       193.22.244.194

  Для всех вопросов ниже можно нажать Enter, чтобы принять значение в [скобках].

  UDP-порт для VPN: число от 1 до 65535 [443]:              ← нажми Enter
  Имя первого устройства (только буквы, цифры, дефис; например my-phone) [my-phone]:   ← нажми Enter или введи своё имя
```

### Что вводить на каждом вопросе

| Вопрос | Что ввести | Пример |
|--------|-----------|--------|
| **Публичный IP** | Скрипт определит сам — нажми **Enter**. Только если не определил, введи IP сервера. | `193.22.244.194` |
| **UDP-порт** | **Число** от 1 до 65535. Лучше оставить 443 (нажми Enter): он маскируется под HTTPS. | `443` |
| **Имя первого устройства** | Короткое имя только из букв, цифр и дефиса. Без пробелов, без слешей, без расширений. | `my-phone` |

### ❌ Чего НЕ делать на промптах

- ❌ Не вводи **путь к файлу** (`/root/vpn-clients/xxx.conf`) — на эти вопросы ждут число (порт) или имя (устройства), не путь.
- ❌ Не пиши имена с расширениями (`my-phone.conf`) — расширение скрипт добавит сам.
- ❌ Не пиши имена с пробелами (`my phone`) — используй дефис (`my-phone`).

Если ошибся — скрипт упадёт с понятным сообщением. Просто запусти его ещё раз (см. ниже).

### Что произойдёт дальше

Через ~1 минуту увидишь:
1. Сообщения об установке пакетов
2. Создание ключей и конфига
3. Запуск туннеля
4. **QR-код прямо в терминале** — твой первый клиент
5. Итоговую сводку с IP сервера и командами для управления

QR можно сразу сканировать камерой телефона из приложения **AmneziaVPN** (не от wireguard.com!).

### Если ошибся в промпте — как перезапустить

Скрипт упал на валидации? Это **не повредило систему** — просто перезапусти:

```bash
sudo bash amneziawg-quickstart/install.sh
```

Если он говорит «Уже установлено» (когда первая попытка успела дойти до конца) — снеси установку и начни заново:

```bash
sudo bash amneziawg-quickstart/uninstall.sh
sudo bash amneziawg-quickstart/install.sh
```

### Если скрипт обновился и хочешь подтянуть свежую версию

```bash
cd ~/amneziawg-quickstart && git pull
```

## Как подключить устройство к VPN

Главный момент для новичка: **конфиги создаются на самом VPS и никуда автоматически не уезжают.** Это безопасно: твой приватный ключ остаётся только у тебя.

После каждой команды `amneziawg-add-peer <имя>` на сервере появляются 2 файла:

```
/root/vpn-clients/<имя>.conf     # текстовый конфиг
/root/vpn-clients/<имя>.png      # QR-картинка с тем же конфигом
```

Плюс в терминале сразу рисуется QR — его можно отсканировать прямо с экрана.

### ❗ Файлы .conf и .png — это не команды

Если в bash просто написать путь к файлу — получишь `No such file or directory` или «Permission denied». **Файлы конфигов не «запускаются» в терминале.** С ними делают три вещи:

| Что нужно | Команда |
|-----------|---------|
| Показать QR в терминале (для скана с телефона) | `qrencode -t ansiutf8 < /root/vpn-clients/<имя>.conf` |
| Посмотреть содержимое (текст) | `cat /root/vpn-clients/<имя>.conf` |
| Скачать на свой компьютер (с локального компа, не с VPS!) | `scp root@<ip>:/root/vpn-clients/<имя>.conf ~/Downloads/` |

Дальше — как именно забрать конфиг на устройство.

### 📱 iPhone / Android — самый простой путь

Никаких файлов скачивать не нужно. Просто:

1. Поставь приложение **AmneziaVPN**:
   - iPhone: [App Store](https://apps.apple.com/app/amneziavpn/id1600529900)
   - Android: [Google Play](https://play.google.com/store/apps/details?id=org.amnezia.vpn)
2. Открой → плюсик в углу → **Сканировать QR-код**.
3. Наведи камеру телефона на QR прямо в окне SSH-терминала (на ноуте/компе, с которого зашёл на VPS).
4. Нажми «Подключить».

Готово. Проверь: открой [2ip.ru](https://2ip.ru) с телефона — там должен быть IP твоего VPS.

**Если QR проскролилось вверх** и его не видно — пересмотри так:

```bash
qrencode -t ansiutf8 < /root/vpn-clients/my-phone.conf
```

### 💻 Windows — нужен файл .conf

Поставь приложение **AmneziaVPN** для Windows: <https://amnezia.org/downloads>

Файл `my-laptop.conf` с VPS нужно забрать на свой компьютер. Два варианта:

**Вариант А — встроенный scp (Windows 10/11).**

Открой PowerShell **на своём компьютере** (не в SSH-сессии VPS!) и выполни:

```powershell
scp root@<ip-сервера>:/root/vpn-clients/my-laptop.conf $env:USERPROFILE\Downloads\
```

Файл появится в `C:\Users\<твой-юзер>\Downloads\my-laptop.conf`.

**Вариант Б — WinSCP (с интерфейсом, без команд).**

1. Скачай WinSCP: <https://winscp.net/eng/download.php>
2. Открой → новое подключение → введи IP сервера, логин `root`, пароль (или ключ).
3. Слева — твой компьютер, справа — сервер. На сервере зайди в папку `/root/vpn-clients/`.
4. Перетащи `my-laptop.conf` в левую панель (например, на Рабочий стол).

После того как файл у тебя на компе:
- Открой AmneziaVPN → **Импорт конфигурации** → выбери `my-laptop.conf`.
- Нажми «Подключить».

### 🍎 macOS — через scp

Поставь приложение **AmneziaVPN** для macOS: <https://amnezia.org/downloads>

Открой Terminal **на своём Маке** (Cmd+Space → «Terminal»):

```bash
scp root@<ip-сервера>:/root/vpn-clients/my-laptop.conf ~/Downloads/
```

Файл будет в `~/Downloads/my-laptop.conf`.

Дальше: AmneziaVPN → **Импорт конфигурации** → выбери файл → «Подключить».

### 🐧 Linux desktop

То же самое что и на macOS:

```bash
scp root@<ip-сервера>:/root/vpn-clients/my-laptop.conf ~/
```

AmneziaVPN → импорт → подключить.

## ⚠️ Важное про безопасность

Файл `.conf` (и QR) содержит **приватный ключ** твоего устройства. Кто угодно с этим файлом сможет подключиться к твоему VPN под видом этого устройства.

**Никогда не делай так:**
- ❌ Не пуши `.conf` или `.png` в GitHub. `.gitignore` в этом репо это блокирует, но не отключай его.
- ❌ Не выкладывай QR-картинку в публичные чаты, на форумы, в социальные сети.
- ❌ Не пересылай `.conf` себе по обычной почте (gmail/mail.ru). Провайдер сохранит копию.

**Можно так:**
- ✅ Сканировать QR прямо с экрана своего терминала (никуда не сохраняется).
- ✅ Скачать через `scp` или WinSCP — это шифрованное соединение.
- ✅ Если очень нужно переслать — Telegram secret chat или Signal. Сразу удали после импорта.

Если случайно засветил `.conf` — удали устройство и создай заново:

```bash
sudo amneziawg-remove-peer my-laptop
sudo amneziawg-add-peer my-laptop
```

Новый ключ → старый файл становится бесполезен.

## Управление устройствами

### Добавить ещё устройство

```bash
sudo amneziawg-add-peer ноут-жены
sudo amneziawg-add-peer ipad
sudo amneziawg-add-peer друг-петя
```

Каждый раз создаётся новый `.conf` файл и QR в `/root/vpn-clients/`. IP в подсети `10.9.0.0/24` подбирается автоматически (следующий свободный).

### Удалить устройство

```bash
sudo amneziawg-remove-peer ipad
```

Активные сессии остальных пиров не разрываются. Файлы `/root/vpn-clients/ipad.*` удаляются автоматически.

### Посмотреть статус

```bash
sudo awg show
```

Покажет всех подключённых, последний handshake, объём трафика по пирам.

### Посмотреть QR для уже созданного устройства

```bash
qrencode -t ansiutf8 < /root/vpn-clients/<имя>.conf
```

### Скачать ещё раз .conf на свой компьютер

```bash
scp root@<ip-сервера>:/root/vpn-clients/<имя>.conf ~/Downloads/
```

## Обычный клиент WireGuard не подойдёт

Важно: для подключения нужно именно приложение **AmneziaVPN**, а не «WireGuard» от wireguard.com. Они выглядят похоже, но обычный WireGuard не понимает параметры обфускации `Jc/S1/H1` — попытается подключиться и получит ошибку.

Где скачать AmneziaVPN: <https://amnezia.org/downloads>

## Что под капотом

Скрипт `install.sh` делает то, что описано в [HOW-IT-WORKS.md](HOW-IT-WORKS.md). Если хочешь понимать что у тебя на сервере — почитай. Все действия легко повторить руками.

## Проблемы

См. [TROUBLESHOOTING.md](TROUBLESHOOTING.md). Самые частые:

- **QR отсканировал, но интернета нет** → у провайдера VPS закрыт `443/UDP`. Открой порт в панели хостера или поменяй на другой.
- **Подключения нет совсем** → проверь firewall провайдера VPS, открыт ли там UDP.
- **На Windows/Mac не импортится** → проверь что ставишь именно **AmneziaVPN**, не WireGuard.

## Удалить всё с сервера

```bash
sudo bash amneziawg-quickstart/uninstall.sh
```

## Безопасность установки

- Все ключи генерятся прямо на твоём VPS. Никто (включая разработчиков этого скрипта) их не видит.
- Параметры обфускации (`H1..H4`) у каждого сервера свои — между разными установками сети не пересекаются.
- Логов трафика **нет**. AmneziaWG ничего не логирует помимо стандартного systemd-журнала (handshake-события).
