# AmneziaWG Quickstart

Свой VPN на своём VPS за 5 минут. Без подписок, без посредников, без логов.

**AmneziaWG** = WireGuard с маскировкой. Под DPI выглядит как случайный UDP-шум на 443-м порту (тот же, что у HTTPS). Работает там, где обычный WireGuard блокируется: РФ, Иран, Китай.

---

> 🆕 **Никогда не работал с серверами и терминалом?** Открой **[📘 Полную инструкцию для новичков (GUIDE.md)](GUIDE.md)** — там каждый шаг разжёван: как купить VPS, как подключиться по SSH, что такое локальный терминал vs SSH-сессия, что делать при типовых ошибках. Со скриншотами.
>
> Ниже — компактная версия для тех, кто уже понимает что такое SSH и `scp`.

## Что получится

- Личный VPN на твоём VPS, доступный с любых устройств.
- Поддержка iPhone, Android, Windows, macOS, Linux.
- Управление 1 командой: `sudo amneziawg-add-peer <имя>` для нового устройства.

## Что понадобится

1. **VPS на Ubuntu 22.04 или 24.04.** Самый дешёвый: 1 CPU / 1 GB RAM / 10 GB. Хостеры: Hetzner (~$4/мес), Aeza, FirstByte.
2. **Локация — где нужен интернет.** Германия, Нидерланды, США, Финляндия. **Не в РФ.**
3. **SSH-доступ к VPS под `root`** (логин и пароль или ключ — у хостера в письме после оплаты).

## Шаг 1. Установка на VPS

### 1.1. Подключись к серверу

Открой терминал (Windows: PowerShell или Терминал; macOS/Linux: Terminal). Введи:

```bash
ssh root@193.22.244.194
```

**Замени `193.22.244.194` на реальный IP твоего сервера** (он в письме от хостера). Угловые скобки не нужны. Введи пароль когда спросит — буквы при вводе не отображаются, это норма.

### 1.2. Запусти установку — 3 команды по очереди

После того как зашёл на сервер, скопируй и вставь эти 3 команды (по одной):

```bash
apt update && apt install -y git
```

```bash
git clone https://github.com/serenkiy-commits/amneziawg-quickstart.git
```

```bash
sudo bash amneziawg-quickstart/install.sh
```

### 1.3. Ответь на 3 вопроса

Скрипт спросит 3 вещи. **На все можно просто нажать Enter** — этого достаточно для рабочей установки.

| Вопрос | Что ввести | Пример |
|--------|-----------|--------|
| Публичный IP сервера | **Enter** (определит сам) | `193.22.244.194` |
| UDP-порт | **Enter** (будет 443, маскируется под HTTPS) | `443` |
| Имя первого устройства | **Enter** (будет `my-phone`) или своё имя латиницей | `my-iphone` |

**⚠️ Важно про имя устройства:**
- Только английские буквы, цифры и дефис.
- Без пробелов, без слешей `/`, без расширений `.conf`.
- ✅ `my-phone`, `iphone15`, `wife-laptop`
- ❌ `my phone`, `мой-телефон`, `phone.conf`, `/root/vpn-clients/phone`

### 1.4. Что произойдёт за следующую минуту

Скрипт сам установит пакеты, сгенерит ключи, поднимет туннель и **нарисует QR-код прямо в терминале**. В конце увидишь сводку:

```
═══════════════════════════════════════════════
  ✓ Готово
═══════════════════════════════════════════════

  Сервер:           193.22.244.194:443 (UDP)
  Интерфейс:        awg0
  Подсеть:          10.9.0.0/24

  Добавить устройство:  sudo amneziawg-add-peer <имя>
  Удалить устройство:   sudo amneziawg-remove-peer <имя>
  Статус туннеля:       sudo awg show
```

Не закрывай это окно — QR прямо над сводкой, сейчас будем его сканировать с телефона.

### Что-то пошло не так?

- **Скрипт упал с «Невалидный порт» или «Невалидное имя»** → ты ввёл не то. Не страшно, ничего не сломано. Просто запусти ещё раз: `sudo bash amneziawg-quickstart/install.sh`
- **Скрипт говорит «Уже установлено»** → значит первая попытка прошла. Можно сразу пользоваться (Шаг 2).
- **Хочешь начать с нуля** → `sudo bash amneziawg-quickstart/uninstall.sh`, потом установка заново.

## Шаг 2. Подключить телефон (1 минута)

Это самый быстрый путь. На телефоне:

1. **Поставь приложение AmneziaVPN** (см. таблицу выше). Это единственное приложение, которое поддерживает AmneziaWG — обычный WireGuard НЕ подойдёт.
2. Открой → нажми **«+»** (или «Добавить») → выбери **«Сканировать QR-код»**.
3. Наведи камеру телефона на QR-код, который скрипт нарисовал в окне терминала на компьютере.
4. Нажми **«Подключить»**.

**Проверка:** открой [2ip.ru](https://2ip.ru) на телефоне. Должен показать IP твоего сервера, а не родной.

### Если QR прокрутился вверх и не виден

Покажи его заново на сервере:

```bash
qrencode -t ansiutf8 < /root/vpn-clients/my-phone.conf
```

(подставь имя своего устройства вместо `my-phone`).

## Шаг 3. Подключить компьютер (Windows / macOS / Linux)

### 3.1. На сервере — создай отдельный конфиг

Один конфиг = одно устройство. Для ноута создай новый:

```bash
sudo amneziawg-add-peer my-laptop
```

Скрипт создаст файл `/root/vpn-clients/my-laptop.conf` и нарисует ещё один QR (для ноута он не нужен).

### 3.2. На своём компьютере — скачай этот файл

⚠️ **Это делается НЕ на сервере, а на твоём компе.** Открой новое окно терминала на своём компе. Если ты сейчас в SSH-сессии (промпт `root@...:~#`), сначала выйди:

```bash
exit
```

Промпт должен стать `PS C:\Users\<юзер>>` (Windows) или `<юзер>@MacBook ~ %` (macOS) — это значит ты на своём компе.

**Windows (PowerShell):**

```powershell
scp root@193.22.244.194:/root/vpn-clients/my-laptop.conf $env:USERPROFILE\Downloads\
```

**Замени `193.22.244.194` на IP своего сервера** (без угловых скобок). Файл появится в `C:\Users\<твой-юзер>\Downloads\my-laptop.conf`.

> **Без командной строки (Windows):** поставь [WinSCP](https://winscp.net/eng/download.php), подключись к серверу (IP, логин `root`, пароль), зайди в папку `/root/vpn-clients/`, перетащи `my-laptop.conf` на свой рабочий стол.

**macOS / Linux:**

```bash
scp root@193.22.244.194:/root/vpn-clients/my-laptop.conf ~/Downloads/
```

(IP замени на свой.)

### 3.3. Импортируй в приложение

1. Если ещё не скачал — поставь **AmneziaVPN** для своей ОС: <https://amnezia.org/downloads>
2. Открой → **«Импорт конфигурации»** → **«Из файла .conf»** → выбери `my-laptop.conf` из Downloads.
3. Нажми **«Подключить»**.

> ⚠️ **Качай только AmneziaVPN** (буква «A» на голубом фоне). Обычный WireGuard (зелёный дракон, от wireguard.com) **не подойдёт** — он не понимает обфускацию AmneziaWG. Иконки разные, но клиенты часто путают.

## Команды управления (на сервере)

| Что нужно | Команда |
|-----------|---------|
| Добавить устройство | `sudo amneziawg-add-peer <имя>` |
| Удалить устройство | `sudo amneziawg-remove-peer <имя>` |
| Кто подключён сейчас | `sudo awg show` |
| Показать QR заново | `qrencode -t ansiutf8 < /root/vpn-clients/<имя>.conf` |
| Скачать конфиг на свой комп (выполнять на локальном компе!) | `scp root@<IP-сервера>:/root/vpn-clients/<имя>.conf ~/Downloads/` (замени `<IP-сервера>` и `<имя>` на свои, скобки убери) |
| Обновить скрипты | `cd ~/amneziawg-quickstart && git pull` |
| Удалить весь VPN | `sudo bash amneziawg-quickstart/uninstall.sh` |

## ❗ Безопасность — важно

Файл `.conf` (и его QR-картинка `.png`) содержат **приватный ключ устройства**. Кто угодно с этим файлом подключится к твоему VPN от твоего имени.

**Никогда:**
- ❌ Не пушь `.conf` или `.png` в GitHub (даже в приватный репо).
- ❌ Не выкладывай QR в публичные чаты, на сайты, в социальные сети.
- ❌ Не пересылай по обычной почте (Gmail, Mail.ru — провайдер хранит копию).

**Можно:**
- ✅ Сканировать QR прямо с экрана своего терминала (никуда не сохраняется).
- ✅ Скачивать через `scp` или WinSCP (шифрованный канал).
- ✅ Telegram Secret Chat / Signal — если нужно срочно переслать.

**Если случайно засветил конфиг** — удали устройство и создай заново, ключ сменится:

```bash
sudo amneziawg-remove-peer my-laptop
sudo amneziawg-add-peer my-laptop
```

Старый файл становится бесполезен.

## Частые недопонимания

**«Я ввёл путь к файлу в промпт скрипта — он упал»**
Скрипт спрашивает либо **число** (порт), либо **имя** (короткое, без слешей). Пути к файлам вводить не нужно — скрипт сам знает где что лежит.

**«Я набрал `/root/vpn-clients/my-phone.png` в терминале — пишет No such file»**
Файлы `.conf` и `.png` — это **просто файлы**, не команды. В bash их не запускают. Что с ними делать:
- Показать QR: `qrencode -t ansiutf8 < /root/vpn-clients/my-phone.conf`
- Посмотреть текст: `cat /root/vpn-clients/my-phone.conf`
- Скачать на свой комп: команда `scp` с локального компа (см. Шаг 3.2)

**«Ввёл scp в терминале, ничего не качается / ругается на команду»**
Две частые причины:
1. Ты ввёл команду **внутри SSH-сессии VPS** (промпт `root@что-то:~#`), а нужно — **на своём компе** в новом окне PowerShell/Terminal (промпт `PS C:\Users\...>` или `user@MacBook ~ %`). Выйди из SSH: `exit`.
2. Ты оставил **угловые скобки** в команде: `root@<193.22.244.194>` вместо `root@193.22.244.194`. Скобки `<>` в инструкции означают «вставь сюда своё значение», их нужно убрать.

**«Установил, подключился, но 2ip.ru показывает родной IP»**
Туннель есть, но трафик идёт мимо. На клиенте не активирован full-tunnel. Проверь что в `.conf` стоит `AllowedIPs = 0.0.0.0/0` (а не `10.9.0.0/24`).

**«Подключаюсь — handshake не происходит»**
У провайдера VPS закрыт UDP-порт. Открой в панели хостера правило для **UDP 443** (или того порта, который ты выбрал).

Подробнее: [TROUBLESHOOTING.md](TROUBLESHOOTING.md).
